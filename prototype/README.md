# 宠拍 AI 原型 · Prototype

基于以下两份 PRD 实现的可交互 HTML 原型：

- `../需求详情-首页.md`
- `../需求详情-图片生成.md`

## 包含页面

| 文件 | 说明 |
|---|---|
| `index.html` | 首页：顶部栏 + 左侧导航 + Hero（3 图片卡片 + 3 视频按钮）+ 爆款图片/视频/模特/场景 4 行横滚 |
| `generate.html` | 图片生成工作台：顶部栏 + 左侧配置面板（上传/宠物场景Tab/提示词/吸底生成按钮）+ 右侧结果区（空态/Loading/2 图对比） |
| `styles.css` | 全站样式（TDesign 风格，主色 `#0052d9`） |
| `data.js` | Mock 数据：爆款图片/视频、10 个宠物模特、8 个场景、结果图池 |
| `home.js` | 首页交互逻辑 |
| `generate.js` | 工作台交互逻辑（上传/选择/生成/预览/登录弹窗） |

## 启动方式（任选其一）

```bash
# 方式 1：直接双击 index.html （部分浏览器对本地 file:// 有限制）
open index.html

# 方式 2：起一个本地静态服务器（推荐）
cd /Users/zonyiac/Documents/产品/aigc-prd/prototype
python3 -m http.server 5173
# 然后浏览器访问 http://localhost:5173
```

## 核心交互路径

1. 首页 → 点击 **宠物模特卡片 hover 出的「制作图片」** → 跳转 `generate.html?petId=xxx`，工作台自动预选该模特
2. 首页 → 点击 **Hero 食品/居家/出行大卡片** → 跳转 `generate.html?category=food`
3. 工作台 → **上传商品图**（支持拖拽）→ 选宠物 → 切到场景 Tab 选场景 → **开始生成**按钮点亮
4. 点击「开始生成」→ 首次未登录触发**登录弹窗** → 登录后进入 5 秒 Loading（真实场景 30s）
5. 生成完成 → 右侧展示 **2 张对比图** → hover 显示 预览/下载/收藏 按钮
6. 点击「重新生成」→ 复用配置再扣 2 积分
